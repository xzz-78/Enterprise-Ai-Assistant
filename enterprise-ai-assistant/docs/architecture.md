# 架构设计文档

## 系统架构

Enterprise AI Assistant 采用前后端分离的微服务架构，主要包含以下几个核心模块：

### 1. 后端服务 (FastAPI)
- **API 层**: 处理 HTTP 请求，参数验证，响应格式化
- **业务逻辑层 (Services)**: 核心业务逻辑实现
- **数据访问层 (Models)**: 数据库 ORM 模型
- **数据验证层 (Schemas)**: Pydantic 数据模型
- **核心配置 (Core)**: 数据库连接、安全认证、配置管理

### 2. AI 服务
- **向量存储 (FAISS)**: 文档向量存储与相似度检索
- **Embedding 服务**: 文本向量化
- **LLM 集成**: 大语言模型调用
- **RAG 管线**: 检索增强生成流程

### 3. 前端应用 (React)
- **页面层 (Pages)**: 各功能页面
- **组件层 (Components)**: 可复用 UI 组件
- **服务层 (Services)**: API 请求封装
- **状态管理**: React Hooks + Context

## 技术选型理由

### 后端框架：FastAPI
- 高性能：基于 Starlette 和 Pydantic
- 自动生成 API 文档（Swagger UI/ReDoc）
- 原生支持异步
- 类型安全

### 数据库：MySQL
- 成熟稳定，企业级应用首选
- 支持事务、索引等高级特性
- 社区生态丰富

### ORM：SQLAlchemy 2.0
- Python 生态最成熟的 ORM
- 功能强大，支持复杂查询
- 类型安全

### 认证方案：JWT
- 无状态，易于扩展
- 前后端分离架构的标准选择
- 支持跨域认证

### AI 框架：LangChain
- 统一的 LLM 接入接口
- 丰富的文档处理工具链
- 支持多种向量数据库

### 向量数据库：FAISS
- Facebook 开源，性能优秀
- 本地部署，数据安全
- 适合中小型项目

### 前端框架：React + TypeScript
- 组件化开发，代码复用性高
- TypeScript 提供类型安全
- 生态丰富，社区活跃

### 构建工具：Vite
- 极速的开发服务器启动
- 热更新速度快
- 优化的生产构建

### 样式方案：TailwindCSS
- 原子化 CSS，减少样式文件体积
- 开发效率高
- 易于定制主题

### 数据可视化：Recharts
- 基于 React 的图表库
- 声明式 API，易于使用
- 性能优秀

## 数据流

### 用户认证流程
1. 用户提交用户名密码
2. 后端验证凭据，生成 JWT Token
3. 前端存储 Token，后续请求携带
4. 后端验证 Token，获取用户信息

### 文档上传流程
1. 用户上传文档文件
2. 后端保存文件到磁盘
3. 解析文档内容（PDF/TXT）
4. 文本切分（Chunking）
5. 生成 Embedding 向量
6. 存入 FAISS 向量索引
7. 记录文档元数据到数据库

### RAG 问答流程
1. 用户输入问题
2. 问题向量化
3. FAISS 相似度检索，获取 Top-K 相关文档
4. 构建 Prompt（上下文 + 问题）
5. 调用 LLM 生成回答
6. 返回答案给用户

### 报告生成流程
1. 获取指定时间范围的销售数据
2. 计算统计指标（总额、均值、增长率等）
3. 格式化数据供 AI 分析
4. 调用 LLM 生成分析内容
5. 调用 LLM 生成改进建议
6. 组装完整报告返回

## 扩展性设计

### 水平扩展
- 无状态 API 服务，可多实例部署
- 向量数据库可迁移到分布式方案（Milvus/Pinecone）
- 数据库可主从读写分离

### 功能扩展
- 插件化的文档解析器
- 可配置的 LLM 提供商
- 模块化的业务服务
